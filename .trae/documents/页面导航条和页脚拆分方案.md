# 页面导航条和页脚拆分方案

## 1. 拆分目标
将所有JSP页面中重复的导航条和页脚代码拆分到独立的公共文件中，实现代码复用和统一管理。

## 2. 拆分方案

### 2.1 创建公共文件
- **header.jsp**：包含导航条代码，处理不同角色的导航逻辑
- **footer.jsp**：包含页脚代码，所有页面共用
- **base.jsp**：包含基础HTML结构、CSS和JS引用（可选，用于简化页面结构）

### 2.2 实现方式
- 使用JSP的`<%@ include file="" %>`指令包含公共部分
- 在header.jsp中保留完整的导航条逻辑，包括角色判断
- 确保CSS和JS引用的路径正确

### 2.3 页面改造
- 每个JSP页面只保留核心内容部分
- 在页面顶部引入header.jsp
- 在页面底部引入footer.jsp
- 移除每个页面中重复的CSS和JS引用（如果使用base.jsp）

## 3. 具体实现步骤

### 3.1 创建公共文件
1. 在`jsp`目录下创建`common`文件夹
2. 创建`common/header.jsp`：包含导航条代码
3. 创建`common/footer.jsp`：包含页脚代码
4. 可选：创建`common/base.jsp`：包含基础HTML结构

### 3.2 修改header.jsp
- 保留完整的导航条代码
- 确保角色判断逻辑正确
- 确保所有链接使用正确的上下文路径

### 3.3 修改footer.jsp
- 包含简单的页脚代码
- 确保格式一致

### 3.4 改造现有页面
- 从首页`index.jsp`开始改造
- 依次改造其他页面，包括：
  - 用户登录注册页面
  - 医生相关页面
  - 患者相关页面
  - 管理员相关页面
  - 错误页面

## 4. 预期效果
- 所有页面的导航条和页脚实现统一管理
- 减少代码重复，提高维护性
- 便于后续统一修改导航样式和内容
- 页面结构更加清晰，只包含核心业务逻辑

## 5. 注意事项
- 确保所有路径引用正确，特别是CSS和JS文件
- 注意JSP include指令的使用方式
- 确保不同角色的导航显示正确
- 测试所有页面的显示效果，确保没有样式错乱
